High-performance CLI tool to calculate geographic distances between points in CSV files using the Haversine formula.

Open Source Project github.com/ethal/ektaon

Batch Processing

Efficiently process massive CSV files with thousands of A ‚Üí B coordinates in seconds.

Global Formats

Supports Decimal Degrees (DD), Degrees-Minutes-Seconds (DMS), and Decimal Minutes (DDM).

Strict Validation

Choose between Permissive mode for bulk processing or Strict mode for data integrity.

Technical Specifications

Input Requirements

The CSV must include headers: name_a, lat_a, lon_a, name_b, lat_b, lon_b.

Coordinate Systems
  • DD: 48.858056 (Decimal)
  • DMS: 48¬∞51'29.6"N (Secs)
  • DDM: 48¬∞51.4'N (Dec. Mins)

Output Data

Ektaon enriches your data with:

  • Distances in km and miles
  • WGS84 Normalized coordinates
  • DMS conversion for all points
  • Nearly-identical flags (11cm tolerance)

CLI Usage

Example Command 
# Standard run
ektaon --input points.csv --output results.csv --input-format=ddm

# Strict mode (stops on first invalid line)
ektaon -i input.csv -o output.csv -f dms --strict

Build from source

Quick Start 
# 1. Clone & Enter
git clone https://github.com/ethal/ektaon.git
cd ektaon

# 2. Build optimized binary
cargo build --release

# 3. Use it from anywhere
./target/release/ektaon --input dd_edge_case.csv --output out.csv
